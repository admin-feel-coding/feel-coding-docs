# Discovery Summary Template (D1–D6)

Use this template to document the Decide phase outputs. It should be shareable with the client.

---

# Discovery Summary — [Project Name]

**Date:** YYYY-MM-DD
**Owner:** [Name]
**Status:** Draft | Reviewed | Approved

## D1 — Problem & Outcomes
- Problem statement (1–2 sentences)
- Primary users and decision-makers
- Success metrics
- Constraints (time, budget, security, compliance)

## D2 — Requirements
- Core user flows (happy path)
- Roles and permissions
- Integrations
- Non-functional requirements (security, performance, availability)

## D3 — Solution Shaping
- High-level architecture sketch (link to diagram)
- Key tradeoffs (for example, REST vs GraphQL)
- Top technical risks

## D4 — MVP & Phases
- Phase 1 (must-have)
- Phase 2/3 (nice-to-have)
- Explicit out-of-scope items

## D5 — Estimation
- Range estimate (min / likely / max)
- Unknowns and spikes needed
- Buffers for risk

## D6 — Proposal-Ready Plan
- Milestones and acceptance criteria
- Assumptions and exclusions
- Change-control rule

## Outputs
- Discovery Summary (this document)
- Architecture sketch
- Phased roadmap
- Hour estimates per phase
